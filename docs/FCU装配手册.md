> **版本**: v1.1 NFC &emsp;|&emsp; **平台**: Arduino Nano (ATmega328P) &emsp;|&emsp; **适用电池**: 3S~6S LiPo  
> **最后更新**: 2026-05

## 一、项目概述

VibeFire FCU（Fire Control Unit）是一款基于 Arduino Nano 的电子火控系统，专为电动软弹/水弹 AEG 发射器设计。它用一个闭环的霍尔传感器检测齿轮位置，配合双 MOSFET 驱动和主动刹车，实现每发必归位、预压弹射和可控射速。内置 OLED 屏幕和三按钮菜单，无需电脑即可在现场调节所有参数。支持 MFRC522 NFC 模块读取弹匣标签，自动装弹并统计余弹。

**核心功能：**

- **闭环归位**：霍尔传感器检测齿轮归位点，保证每发完整循环
- **预压延迟**：归位后延迟刹车，提前压缩弹簧，消除扳机延迟
- **扳机不中断**：扣下扳机后必须完成当前循环，松手不打断
- **4 档快慢机**：保险 / 单发 / 双行程 / 连发，通过 2 个霍尔传感器编码
- **双行程模式**（Binary Trigger）：扣下击发一发，松开击发第二发
- **x 连发模式**：每次扣扳机击发 N 发（1~10 可调），连发中途再次扣扳机重置计数
- **主动刹车**：双 MOSFET 电路，归位后短路电机实现精准制动
- **OLED 菜单系统**：0.96 寸蓝/黄双色屏，三按钮操作，存储参数到 EEPROM
- **NFC 弹匣管理**：弹匣上贴 NFC 标签，换匣自动读取出厂容量，实时计弹
- **多重保护**：欠压锁定、堵转紧急停机、MOS 死区互锁、看门狗复位

**系统架构（简化）：**

```
  6S LiPo ──┬── XT60 ── 电机驱动回路 (大电流)
            │      ├── 电机+ ─── 电机绕组 ─── 电机- ─── 主MOS(D) ─── [S]── GND
            │      └── 电机+ ─── 刹车MOS(D) ─── [S]── GND
            │
            └── Mini-360 降压 (6S→5V)
                   ├── Arduino Nano 5V
                   ├── OLED I2C
                   ├── 归位霍尔 (D3)
                   ├── 快慢机霍尔×2 (D4, D5)
                   ├── MFRC522 NFC SPI
                   └── 分压检测 (A0)
```

**目标读者**：DIY 爱好者、水弹/软弹改装玩家，具备基础的焊接和 Arduino 使用经验。

## 四、硬件接线

### 4.0 接线前必读

- **共地原则**：所有模块的 GND 最终都接到电池负极。降压模块输出的 GND = Arduino GND = 电池负极。
- **大电流路径**：电池(+) → 电机(+) → 电机绕组 → 电机(-) → 主MOS(D) → 主MOS(S) → 电池(-)。这段用 **14AWG** 硅胶线。
- **信号线**：霍尔、按钮、I2C 用 22AWG 或杜邦线即可，但要远离电机线（至少 2cm）避免干扰。
- **电容极性**：100μF 电解电容有极性——长脚为正(+)，短脚为负(-)，外壳有负极标记带。接反会爆！

### 4.1 电源模块接线

| Mini-360 引脚 | 连接目标 | 建议线色 |
|--------------|----------|---------|
| IN+ | XT60 公头 红(+) | 红 |
| IN- | XT60 公头 黑(-) | 黑 |
| OUT+ | Arduino 5V + OLED VCC + 霍尔模块 VCC (×3) | 红 |
| OUT- | Arduino GND + OLED GND + 霍尔模块 GND (×3) | 黑 |

> Mini-360 板上有蓝色微调电位器，不要动它——出厂默认 5V 输出。如需验证，上电后用万用表测 OUT+ 对 OUT- 应为 5.0V ± 0.2V。

### 4.2 OLED 屏幕接线 (I2C)

### 4.7 MOSFET 功率电路接线

```
        主驱动 MOSFET (IRLB3034-1)              刹车 MOSFET (IRLB3034-2)
    ┌──────────────────────────┐          ┌──────────────────────────┐
    │                          │          │                          │
    │  G (Pin1,左) ──┬── D6   │          │  G (Pin1,左) ──┬── A1   │
    │                │         │          │                │         │
    │               10KΩ       │          │               10KΩ       │
    │                │         │          │                │         │
    │               GND        │          │               GND        │
    │                          │          │                          │
    │  D (Pin2,中) ─── 电机(-) │          │  D (Pin2,中) ─── 电机(+) │
    │                          │          │                          │
    │  S (Pin3,右) ─── 电池(-) │          │  S (Pin3,右) ─── 电池(-) │
    │                          │          │                          │
    │  TAB (金属背板) = D      │          │  TAB (金属背板) = D      │
    └──────────────────────────┘          └──────────────────────────┘

    IRLB3034 TO-220 引脚定义（面朝自己，引脚向下）：
         ┌─────────┐
         │  IRLB   │  ← 金属背板 = 漏极(D)，内部连通 Pin2
         │  3034   │
         └─────────┘
          │  │  │
          │  │  └── Pin 3: 源极 Source (S)  → GND
          │  └───── Pin 2: 漏极 Drain (D)   → 电机线
          └──────── Pin 1: 栅极 Gate (G)    → Arduino D6 (主) / A1 (刹车)
```

> **关键警告**：主 MOS 和刹车 MOS **绝对不能同时导通**（短路电池！）。软件有双重保护（先关刹车再开主驱，且中间有 dead-time 延时），但仍然建议物理上确认接线正确。

### 4.8 电池电压分压电路接线

```
电池(+) ──── 100KΩ ────┬──── Arduino A0
                       │
                      10KΩ
                       │
                      GND
```

> 分压比：10K / (100K + 10K) = 1/11  
> 6S 满电 25.2V → A0 约 2.29V → ADC 读数约 469  
> 6S 存储 22.2V → A0 约 2.02V → ADC 读数约 413  
> 6S 欠压 19.8V → A0 约 1.80V → ADC 读数约 369

## 五、功率电路焊接指南

功率电路是本项目唯一需要焊接的部分（信号部分全部杜邦线插接）。以下为完整的焊接指导。

### 5.1 焊接准备

1. 准备 5×7cm 双面万能板
2. 确认烙铁头清洁上锡，温度设 350°C（有铅锡丝）或 380°C（无铅锡丝）
3. 给两个 TO-220 散热片涂一薄层导热硅脂（如有），用螺丝或卡扣固定在 MOS 管背面

### 5.2 IRLB3034 引脚识别

```
拆机图/实物图（面朝自己，字朝自己，引脚向下）：

   ┌─────────────┐
   │   IRLB3034  │  ← 金属背板 = D (漏极)
   │   XXXXXXXX  │
   │   XXXXXXXX  │
   └────┬───┬────┘
        │   │
   ┌────┘   └────┐
   │   1   2   3  │  ← 引脚
   └──────────────┘
   
   Pin 1 (左) = G  (栅极 Gate)
   Pin 2 (中) = D  (漏极 Drain)
   Pin 3 (右) = S  (源极 Source)
```

### 5.3 焊接顺序（推荐）

**第 1 步：安置 MOSFET**

- 将两个 IRLB3034 插入万能板，引脚间距兼容标准 2.54mm 孔距
- 两个 MOS 管间距至少留 2cm，方便散热
- 焊接前用万用表蜂鸣档确认各引脚未与相邻孔短路（万能板多孔连通注意）

**第 2 步：焊接栅极下拉电阻**

- 两个 10KΩ 电阻，一端焊到各自 MOS 的 G (Pin1)，另一端焊到 S (Pin3) 或 GND 总线
- 这保证 Arduino 未上电/复位期间 MOS 不会浮空误导通
- **先焊这个**，再焊别的——安全第一

**第 3 步：焊接大电流路径（14AWG）**

```
焊接顺序（按电流流向）：
  GND 总线(铜线) ──→ 主MOS S(Pin3) ──→ 主MOS D(Pin2) ──→ 螺丝端子(电机-)
                    ──→ 刹车MOS S(Pin3) ──→ 刹车MOS D(Pin2) ──→ 螺丝端子(电机+)
```

- 用粗铜线（剪一段 14AWG 硅胶线的铜芯，或拆一段网线铜芯）在万能板上走 GND 总线
- 14AWG 线直接焊到螺丝端子上——螺丝端子一端插线，另一端焊在万能板孔位

**第 4 步：焊接栅极信号线**

- 主 MOS G(Pin1) 焊一根细线 → 引出杜邦线母头 → 接 Arduino D6
- 刹车 MOS G(Pin1) 焊一根细线 → 引出杜邦线母头 → 接 Arduino A1
- 或者直接将 22AWG 线一头焊在 MOS G 脚，另一头剥皮后插入 Nano 排针

**第 5 步：焊接分压电阻**

- 100KΩ 一端接电池正极焊盘（XT60 红出线处），另一端接 Arduino A0
- 10KΩ 一端接 A0，另一端接 GND 总线
- 建议在万能板空闲区域用三个焊点搭好

**第 6 步：焊接滤波电容**

- 100μF/35V 电解：长脚(+)焊电机(+)螺丝端子，短脚(-)焊电机(-)螺丝端子
- 100nF 瓷片：无极性别，直接并联在电机两端
- **100μF 极性绝对不能焊反！** 焊反通电后电容会爆炸。

**第 7 步：焊接 XT60 公头**

- 14AWG 红线焊 XT60 的 + 端（半圆形端）
- 14AWG 黑线焊 XT60 的 - 端（六角形端）
- 热缩管包裹每个焊点
- 红线的另一端接到 MINI-360 的 IN+ 和电机正极总线（分叉）
- 黑线的另一端接到 MINI-360 的 IN- 和电机负极总线（分叉）

### 5.4 焊接后检查清单

| # | 检查项 | 方法 |
|---|--------|------|
| 1 | 所有焊点光亮饱满，无冷焊/虚焊 | 肉眼检查 + 轻拉线头 |
| 2 | 无焊锡桥接导致短路 | 万用表蜂鸣档，相邻焊点不通 |
| 3 | MOS S→GND 导通 | 蜂鸣档测 S(GND) ↔ XT60 黑，应通 |
| 4 | 栅极下拉电阻阻值正确 | 电阻档测 G↔S，应为 10KΩ ± 5% |
| 5 | 100μF 电容极性正确 | 正极接电机+，负极接电机- |
| 6 | 分压网络阻值正确 | 测 100KΩ 和 10KΩ 实际阻值 |
| 7 | 热缩管覆盖所有裸露焊点 | 肉眼检查，无裸露金属 |
| 8 | MOS 散热片安装牢固 | 轻推不晃动 |

## 八、首次开机与校准

### 8.1 首次上电检查清单

| # | 检查项 | 预期 |
|---|--------|------|
| 1 | OLED 显示正常，无花屏/缺行 | 全屏 128×64 完好 |
| 2 | 电压读数正确 | 与万用表实测值误差 <0.3V |
| 3 | 快慢机旋转到各挡位，OLED 模式字样切换 | SAFE → SEMI → BINARY → AUTO |
| 4 | 弹量为 RFID 标签容量（有标签时）或 0（无标签时） | 根据实际情况 |
| 5 | 按钮反应灵敏，无粘连 | 按一次只响应一次 |
| 6 | LED 指示灯常亮（非故障状态） | 板载 LED 或 A3 外接 LED 亮 |

### 8.2 菜单导航基础

| 操作 | 功能 |
|------|------|
| UP 短按 | 光标上移 / 数值增加 |
| DOWN 短按 | 光标下移 / 数值减少 |
| SEL 短按 | 进入子菜单 / 确认 |
| SEL 长按 (>800ms) | 从主界面进入主菜单 / 从任意位置退回主界面 |
| 扳机扣下 | 在菜单中 = 确认/选择 |
| 无操作 8 秒 | 自动退出菜单，保存参数 |

### 8.3 预压延迟调优

预压延迟决定了齿轮到达归位点后，电机继续运转多少毫秒才刹车。目的：让天梯在归位后被额外压缩一段，下一发响应更快。

**调优方法**：

1. 先将预压设为 **0ms**，打几发感受扳机响应
2. 每次 +5ms，再打几发
3. 找到「扳机响应明显变快」但「齿轮不卡顿/电机不堵转」的值
4. 典型值：**15~35ms**（因发射器而异）
5. 超过 50ms 如果感觉不到改善，说明已经过了最佳点

### 8.4 火控模式配置

默认映射（可改）：

| 快慢机档位 | 默认模式 | 可选模式 |
|-----------|---------|---------|
| 第 1 档 (P1) | SAFE (保险) | SAFE / SEMI / BINARY / BURST / AUTO |
| 第 2 档 (P2) | SEMI (单发) | 同上 |
| 第 3 档 (P3) | BINARY (双行程) | 同上 |
| 第 4 档 (P4) | AUTO (连发) | 同上 |

> 进入菜单 → Mode Config → 用 UP/DOWN 选档位 → 扣扳机切换模式 → SEL 短按返回。

**x 连发模式 (BURST)**：选 BURST 后自动进入设置界面，可设 2~10 发。每次扣扳机击发 N 发，中途再次扣扳机重置计数从头开始。

### 8.5 NFC 弹匣标签编程

**方法一：用手机 NFC 写入（推荐，最方便）**

1. 下载 NFC Tools APP（Android/iOS 均有）
2. 打开 APP，选择「写」→「添加记录」→「自定义文本」
3. 输入：`FCUMAG` + 填充（共 10 字节 = 20 个十六进制字符）
4. 格式：`FCUMAG` + 2 字节容量 + 2 字节填充
5. 例如容量 120 发：`46 43 55 4D 41 47 00 00 00 78 00 00`（前 6 字节为 "FCUMAG"，第 9-10 字节为 0x00 0x78 = 120）
6. 将标签贴近手机 NFC 区域，写入

**方法二：用 Arduino + MFRC522 写入**

写一个临时的写入 Sketch，参考 MFRC522 库例程中的 `WriteData`。

> 写入后把标签贴在弹匣底部。插匣时 FCU 自动读取容量并装弹。拔匣后弹量归零，再次插匣重新装载标签容量。

## 九、菜单操作完整指南

### 9.1 主界面显示

```
┌──────────────────────────┐
│ 24.5V  30RPS       OK   │  ← 电压 | 射速(RPS) | 状态(OK/ERR/LOW)
│──────────────────────────│  ← 分割线
│                          │
│         24               │  ← 大字：剩余弹量
│          /30             │  ← 小字：/弹匣容量
│                          │
│ ████████████████░░░░░░   │  ← 弹量进度条
│ SEMI          HOLD SEL   │  ← 当前模式 | 提示文字
└──────────────────────────┘
```

**状态指示**：
- `OK` = 正常待机
- `>>` = 激发中
- `ERR` = 堵转故障（需断电复位）
- `LOW` = 电池欠压
- `EMPTY` = 弹匣有标签但弹量为 0
- `NO MAG` = 未检测到弹匣 NFC 标签

### 9.2 主菜单（5 项）

按 SEL 长按进入：

```
┌──────────────────────────┐
│ ===== MENU =====         │
│                          │
│ > Pre-Comp Delay         │  ← 预压延迟（0-100ms，步进 5ms）
│   Empty Lock             │  ← 空匣锁定开关
│   Mode Config            │  ← 模式映射（4 档 × 5 种模式）
│   Ammo Count             │  ← 弹量调节（0-999，步进 5）
│   Battery Info           │  ← 电池信息（只读）
│                          │
│ TRIG=Sel  SEL=Back       │  ← 操作提示
└──────────────────────────┘
```

### 9.3 子菜单详解

**① Pre-Comp Delay（预压延迟）**

```
┌──────────────────────────┐
│ === PRE-COMP ===         │
│                          │
│ Delay:  30 ms            │  ← 当前值
│                          │
│ ██████████████░░░░░░░░░  │  ← 进度条 (30/100)
│                          │
│ UP +5   DOWN -5          │  ← 短按调值
│ SEL=Back                 │
└──────────────────────────┘
```

范围：0~100ms，步进 5ms。长按 UP/DOWN 会连续增加/减少。

**② Empty Lock（空匣锁定）**

```
┌──────────────────────────┐
│ == EMPTY LOCK ==         │
│                          │
│ Status:  ON              │  ← ON=空匣禁射, OFF=允许无弹激发
│                          │
│ ammo=0 blocks            │  ← 说明文字
│                          │
│ TRIG=Toggle  SEL=Back    │
└──────────────────────────┘
```

扣扳机切换 ON/OFF。设为 ON 时弹量为 0 无法激发；设为 OFF 时空弹也能打（调试用）。

**③ Mode Config（模式映射）**

```
┌──────────────────────────┐
│ === MODE MAP ===         │
│                          │
│ > P1: SAFE               │  ← 第 1 档：保险
│   P2: SEMI               │  ← 第 2 档：单发
│   P3: BINARY             │  ← 第 3 档：双行程
│   P4: BURST x3           │  ← 第 4 档：3 连发
│                          │
│ UP/DN=Slot  TRIG=Chg     │
└──────────────────────────┘
```

- UP/DOWN 移动 `>` 光标选择档位
- 扣扳机切换该档位模式（SAFE → SEMI → BINARY → BURST → AUTO → SAFE...）
- 切换到 BURST 时自动弹出连发数设置

**④ Ammo Count（弹量调节）**

```
┌──────────────────────────┐
│ === AMMO ===             │
│                          │
│ Count:  120              │  ← 当前弹量
│                          │
│ UP +5   DOWN -5          │  ← 步进 5 发
│ SEL=Back                 │
└──────────────────────────┘
```

> 有 NFC 标签时不要手动改弹量——插拔一次弹匣会自动重置为标签容量。手动调节主要用于无 NFC 标签的场景。

**⑤ Battery Info（电池信息）**

```
┌──────────────────────────┐
│ === BATTERY ===          │
│                          │
│ Total:  24.5V            │  ← 总电压
│ Cell:    4.08V           │  ← 每节电芯电压（总/6）
│                          │
│ SEL=Back                 │
└──────────────────────────┘
```

只读页面，无法修改。Cell 电压低于 3.30V 时触发欠压保护（主界面显示 LOW），禁止发射。

## 十、故障排查

| # | 现象 | 可能原因 | 排查方法 |
|---|------|---------|---------|
| 1 | OLED 不亮，无任何显示 | I2C 地址不对；接线松脱；OLED 烧毁 | 用 I2C Scanner 例程扫描地址；测 OLED VCC-GND 应 5V；换一块 OLED 试试 |
| 2 | OLED 花屏或部分行不显示 | I2C 干扰；供电不足 | 检查 Mini-360 输出是否稳定 5V；SDA/SCL 线远离电机线 |
| 3 | 电机不转 | MOS 接线错误；栅极下拉过大；电机线断 | 万用表测主 MOS G 极电压：扣扳机时应 >4V；测 D 极对 GND 电压 |
| 4 | 电机一直转不停 | 归位霍尔未触发；磁铁掉了 | 看霍尔模块红色 LED：磁铁经过时是否亮？检查磁铁是否还在齿轮上 |
| 5 | 刹车无效，齿轮过冲 | 刹车 MOS 烧了；接线错误 | 刹车时测电机两端电压应接近 0V；检查刹车 MOS 是否发热异常 |
| 6 | 刹车太猛，齿轮卡顿 | 预压为 0 却过早刹车 | 可能是归位霍尔位置偏差，微调传感器位置 |
| 7 | MOS 管发烫（>60°C） | 散热不足；栅极驱动不够；短路 | 加散热片+硅脂；测 10K 下拉电阻；测电机电流是否异常大 |
| 8 | 上电瞬间电机抖一下 | 栅极下拉电阻未焊好；Arduino 复位时引脚浮空 | 检查两个 10K 电阻是否一头接 G 一头接 S/GND |
| 9 | 串口输出乱码 | 波特率不匹配 | 确认串口监视器设为 **115200** baud |
| 10 | 堵转误报（STALL!） | STALL_TIMEOUT_MS 太短；齿轮阻力大 | 固件默认 2500ms，大扭力电机可能需要 3000~4000ms |
| 11 | NFC 读不到标签 | 天线被金属遮挡；距离太远；标签类型不匹配 | 确保 NFC 天线远离金属壳体；标签贴近天线 <2cm；用 NTAG215 或 MIFARE Classic 1K |
| 12 | NFC 模块完全不工作 | SPI 接线错误；用 5V 供电烧了 | 测 MFRC522 VCC 应为 3.3V，不是 5V！换一块 RC522 |
| 13 | 快慢机档位识别不准 | 磁铁位置偏移；霍尔去抖不足 | 旋转快慢机，确认磁铁正对传感器时 LED 亮；检查 SELECTOR_DEBOUNCE_MS |
| 14 | 双行程模式松手不击发第二发 | binary_armed 状态被清除；信号丢失 | 检查扳机松手时 D2 是否确实变 HIGH；微动接触是否良好 |
| 15 | 参数断电后丢失 | EEPROM 写入失败 | 检查 `save_eeprom()` 是否在退出菜单时被调用；换一颗 Nano（EEPROM 寿命约 10 万次） |
| 16 | 看门狗复位频繁 | loop() 中有阻塞代码；传感器读取卡死 | 串口监视器看是否有重复的 "FCU v1.1 NFC" 启动画面。有 = 看门狗在复位 |

## 十一、安全警告

> **请逐条阅读并理解。忽视这些警告可能导致设备损坏、人身伤害或火灾。**

### ⚠️ LiPo 电池安全

- **必须使用平衡充电器**充电。严禁用非 LiPo 充电器或直接接直流电源充电。
- **充电时必须有人看管**。LiPo 电池过充会起火。
- **电池鼓包、变形、漏液请立即停止使用**，按当地电池回收规定处置。
- **存放电压**：长期不用请将电池充/放电至 3.80V/cell（6S = 22.8V）存放。
- **不要过放**：单片电压低于 3.00V 会永久损坏电池。本 FCU 在 3.30V/cell 时锁定，但请勿依赖此保护。

### ⚠️ MOSFET 死区保护

- **主驱动 MOS 和刹车 MOS 绝对不能同时导通。** 同时导通 = 电池正负极直接短路，瞬间电流可达数百安培，会导致线路起火、电池爆炸。
- 固件通过软件 dead-time 保护（先关刹车等 80μs 再开主驱）。但这不能替代正确的硬件接线。
- 接线完成后，用万用表蜂鸣档确认：两只 MOS 的 D 极之间不应导通（电机绕组断开时）。

### ⚠️ 眼部保护

- 调试、测试时 **必须佩戴护目镜**。软弹/水弹仍有一定动能，近距离可伤眼。
- 测试场内所有人必须佩戴护目镜，不区分"我只是路过"。

### ⚠️ 发射器安全守则

- **永远假设发射器已上膛/已通电。**
- **不要将发射器指向任何人或动物**，即使你认为保险已打开。
- **扳机纪律**：不准备发射时，手指离开扳机护圈。
- **运输时断开电池**，不要依赖保险档位。

### ⚠️ 法律合规

- 不同地区对仿真枪/软弹发射器/水弹发射器有不同法律规定。请在组装和使用前了解并遵守当地法律。
- 发射器初速、外观、使用场所可能受法律限制。
- **在公共场所展示、携带可能触犯法律。** 请在私人场地或合法射击场使用。
- 本手册仅提供技术指导，用户对合法合规使用负全部责任。

## 十二、可选增强

以下功能非必须，但可以显著提升体验。它们需要额外的硬件模块和固件修改。

### 12.1 蓝牙无线调参 (HC-06)

添加 HC-06 蓝牙模块后，可以通过手机串口 APP 远程调节所有参数，无需按按钮。

- **接线**：HC-06 TX → Nano D0 (RX)，HC-06 RX → Nano D1 (TX)，VCC → 5V，GND → GND
- **注意**：烧录固件时需要断开 HC-06（它与 USB 共用串口），烧完再接上
- **手机 APP**：下载 "Serial Bluetooth Terminal" (Android) 或 "Bluetooth Terminal" (iOS)
- **AT 命令**：可以自定义蓝牙名称和波特率

### 12.2 陀螺仪稳控 (MPU6050)

加装 MPU6050 六轴传感器，可以根据发射器姿态控制发射逻辑。

- **接线**：I2C，与 OLED 共用 SDA/SCL 总线
- **应用**：检测枪口上扬角，超过阈值后暂停连发；快速水平移动时自动切换为单发防止误伤

### 12.3 弹匣插入自动重置

在弹匣井加装一个微动开关，弹匣插入时触发 → 自动将弹量重置为满容量。

- **接线**：微动一端接 Arduino 空闲引脚（如 A6），另一端接 GND
- **逻辑**：微动触发 → `ammo = mag_capacity`（或默认 30）

### 12.4 单体电池电压监控

利用 6S 平衡头的 7 根线（6 节 + 1 地），通过分压网络接到 Arduino 多个模拟引脚，实现对每节电芯的独立监控。

- 需要 6 组分压电阻和 6 个 ADC 通道（Nano 只有 8 个 ADC，够用）
- 可在 OLED 上显示每节电压，精确定位哪节电芯出了问题

### D. EEPROM 存储布局

| 地址 | 数据 | 字节数 | 默认值 |
|------|------|--------|--------|
| 0~1 | 预压延迟 (ms) | 2 | 0x0000 (0ms) |
| 2~3 | 弹量 | 2 | 0x001E (30) |
| 4~7 | 模式映射 (4 字节) | 4 | SAFE, SEMI, BINARY, AUTO |
| 8~11 | x 连发数 (4 字节) | 4 | 3, 3, 3, 3 |
| 12 | 空匣锁定 | 1 | 0x01 (ON) |

> EEPROM 寿命约 10 万次。参数只在退出菜单时写入一次，不是每次都写。
