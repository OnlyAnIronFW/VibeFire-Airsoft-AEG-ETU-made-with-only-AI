## 1. 系统简介

### 1.1 AutoTrigger 是什么？

AutoTrigger 是一个安装在玩具枪（海绵软弹/水弹）上的计算机视觉系统，它能够：

```
                  ┌──────────────┐
 目标检测  ──────▶│  YOLOv5n AI  │  识别人物，实时追踪
                  │  (RK3566 NPU) │
                  └──────┬───────┘
                         │
                  ┌──────▼───────┐
 测距+弹道  ──────▶│  激光测距 +  │  计算距离 + 弹道下坠补偿
                  │  空气阻力模型 │
                  └──────┬───────┘
                         │
                  ┌──────▼───────┐
 瞄准显示  ──────▶│  圆形OLED屏  │  显示补偿后的瞄准点
                  │  (240×240)   │
                  └──────┬───────┘
                         │
                  ┌──────▼───────┐
 自动激发  ──────▶│  GPIO 信号   │  瞄准点对齐时自动输出激发信号
                  └──────────────┘
```

### 1.2 核心参数

| 参数 | 数值 |
|------|------|
| 有效交战距离 | 40-60m |
| 目标检测距离 | ≥40m |
| 检测帧率 | 30 FPS |
| 端到端延迟 | ~38-52ms |
| 系统功耗 | 5W |
| 续航时间 | ~10小时 (10000mAh) |
| 整机重量 | ~180g |
| 总成本 | ¥755-785 |

### 1.3 工作原理（30秒理解）

```
每一帧（33ms）的处理流程：

  摄像头采集 ──▶ AI识别人物 ──▶ 激光测距 ──▶ 计算弹道补偿
       │              │              │              │
       │           找到人?       距离多少?      子弹掉多少?
       │              │              │              │
       └──────────────┴──────────────┴──────▶ 屏幕上显示瞄准点
                                                  │
                                          瞄准点=准星? ──▶ 自动开火!
```

**关键概念**：
- 🔴 **十字准星**：枪管指向的位置（屏幕中心固定）
- 🟢 **瞄准点**（小绿点）：系统计算出的"应该瞄哪里"（考虑弹道下坠+目标运动）
- 🎯 **当十字准星对准绿点时**，扣住扳机 → 自动激发

### 3.3 组装步骤

#### 步骤1：焊接主板排针（5分钟）

```
   Radxa Zero 3E 背面 GPIO 孔
   ┌─────────────────────┐
   │ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ │  ← 2排20孔 = 40Pin
   │ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ │
   └─────────────────────┘
            │
     ┌──────▼──────┐
     │  焊接排针     │  ← 2.54mm 排针，公头朝下
     │  (注意垂直)   │
     └─────────────┘
```

1. 将排针插入主板背面GPIO孔
2. 用电烙铁逐个焊接
3. 检查焊点：光滑、饱满、无连锡

#### 步骤2：连接OLED屏幕（10分钟）

```
   OLED 背面引脚：
   ┌────┐
   │VCC │────▶ 主板 5V (Pin 2)
   │GND │────▶ 主板 GND (Pin 6)
   │DIN │────▶ 主板 MOSI (Pin 19)
   │CLK │────▶ 主板 SCLK (Pin 23)
   │DC  │────▶ 主板 GPIO (Pin 18)
   │RST │────▶ 主板 GPIO (Pin 22)
   └────┘
```

1. 剪6段杜邦线（母头），每段约8cm
2. 按接线表焊接到OLED对应焊盘
3. 热缩管保护焊接处
4. 杜邦线另一端插入主板对应GPIO孔

#### 步骤3：连接激光测距（5分钟）

```
   JRT TOF01 接线端：
   ┌──────────────┐
   │ VCC ────────▶│── 主板 5V (Pin 2)
   │ GND ────────▶│── 主板 GND (Pin 6)
   │ TX  ────────▶│── 主板 RX (Pin 10)
   │ RX  ────────▶│── 主板 TX (Pin 8)
   └──────────────┘
```

⚠️ **重要**: TX接RX, RX接TX（交叉接线）

1. 剪4段杜邦线
2. 按图焊接/连接
3. 热缩管保护

#### 步骤4：连接摄像头（2分钟）

```
   主板 CSI 接口
   ┌────────────┐
   │ ▓▓▓▓▓▓▓▓▓ │  ← 15Pin FPC 连接器
   │ ▓▓▓▓▓▓▓▓▓ │
   └────────────┘
        │
   ┌────▼────┐
   │  IMX219 │  ← 软排线直接插入
   │  摄像头  │
   └─────────┘
```

1. 轻轻撬起CSI连接器黑色卡扣
2. 将IMX219软排线**铜触点朝上**插入
3. 按下黑色卡扣锁紧

#### 步骤5：引出触发信号（3分钟）

```
   主板 GPIO Pin 11 ──────▶ 发射器控制板
         │                       │
         │   ┌─────┐             │
         └──▶│ 开关 │────────────┘
             └─────┘
             (可选，用于物理安全切断)
```

1. 杜邦线从Pin 11引出（高电平=激发）
2. GND从Pin 14引出
3. 可选：串联拨动开关作为物理安全锁

#### 步骤6：装入外壳（10分钟）

```
   3D打印外壳结构（剖面）：
   
   ┌─────────────────────────┐
   │        OLED 屏幕         │  ← 屏幕开窗
   │     ┌───────────┐       │
   ├─────┤           ├───────┤
   │     │  主板+散热 │       │
   │     │  (固定在螺柱)│      │  ← 内部腔体
   │     │           │       │
   ├─────┴───────────┴───────┤
   │  皮轨卡扣   │   电池仓   │  ← 底部
   └─────────────────────────┘
          │
    ┌─────▼─────┐
    │  玩具枪皮轨 │  ← 标准20mm皮卡汀尼导轨
    └───────────┘
```

1. 主板用M2.5螺柱固定在外壳底部
2. OLED嵌入屏幕开窗，用热熔胶固定
3. 激光测距对准外壳前方的开孔
4. 摄像头固定在外壳前方（镜头朝外）
5. 合上外壳，拧紧螺丝

#### 步骤7：最终检查

- [ ] USB-C插入充电宝，主板指示灯亮？
- [ ] OLED屏幕点亮，显示十字准星？
- [ ] 所有连接牢固，无松动？
- [ ] 外壳螺丝拧紧？
- [ ] 皮轨卡扣可靠锁定？

## 4. 软件安装

### 4.1 烧录 Armbian 系统到 TF 卡

**需要**：一台Windows/Mac/Linux电脑 + 读卡器 + 32GB TF卡

**步骤**：

1. 下载 Armbian for Radxa Zero 3E：
   ```
   https://www.armbian.com/radxa-zero-3e/
   ```

2. 使用 **balenaEtcher** 或 **Rufus** 烧录镜像到TF卡

3. 烧录完成后，将TF卡插入Radxa Zero 3E

4. 首次启动（通过USB-C连接显示器+键盘，或SSH连接）：
   ```bash
   # 默认用户名/密码：root / 1234
   # 按提示修改密码、创建用户
   ```

### 4.2 安装依赖

SSH登录后执行：

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装编译工具
sudo apt install -y build-essential cmake git

# 安装运行时依赖
sudo apt install -y libopencv-dev libsdl2-dev libgpiod-dev libeigen3-dev

# 加载NPU驱动
sudo modprobe rknpu
echo "rknpu" | sudo tee -a /etc/modules

# 验证NPU驱动
ls /dev/rknpu  # 应该存在

# 重启
sudo reboot
```

### 4.3 编译安装 AutoTrigger

```bash
# 克隆项目
git clone https://github.com/yourname/autotrigger.git
cd autotrigger

# 生成弹道表（Python脚本）
python3 scripts/generate_drop_table.py
# → 输出: tables/drop_table.bin

# 编译
mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j4

# 安装
sudo make install
```

### 4.4 准备 YOLOv5n 模型（待实现）

> ⚠️ **当前状态**: YOLOv5n RKNN 模型转换和实际推理为 v1.1 计划功能。
> v1.0 使用 `YOLOMock` 进行开发验证，真实 RKNN 推理待后续实现。

模型转换需在**电脑**上完成（需要x86环境 + RKNN-Toolkit2）：

```bash
# 安装 RKNN-Toolkit2
pip install rknn-toolkit2

# 下载 YOLOv5n ONNX 模型并转换为 RKNN
# (具体转换脚本见 scripts/ 目录，后续版本提供)
# python3 scripts/convert_yolov5n.py --input yolov5n.onnx --output models/yolov5n.rknn
```

### 4.5 设置开机自启

```bash
# 创建 systemd 服务
sudo tee /etc/systemd/system/autotrigger.service << 'EOF'
[Unit]
Description=AutoTrigger Auto-Aim System
After=network.target

[Service]
Type=simple
User=radxa
WorkingDirectory=/home/radxa/autotrigger
ExecStart=/usr/local/bin/autotrigger --v0 70
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

# 启用
sudo systemctl enable autotrigger
sudo systemctl start autotrigger

# 查看状态
sudo systemctl status autotrigger
```

## 5. 校准流程

> ⚠️ **校准是保证精度的关键步骤**，每次换枪或重新安装后必须校准。
> 校准一次约需 **5分钟**，建议在**靶场环境**下完成。

### 5.1 相机内参校准（一次性）

⚠️ **当前**: 校准程序为库代码 (`calibrate.cpp`)，需自行编写调用程序或使用 OpenCV 独立完成。

**步骤**：
1. 打印棋盘格（A4, 8×6）
2. 在不同角度拍摄10-15张棋盘格照片
3. 使用 OpenCV 校准：
   ```bash
   # 使用 OpenCV 自带工具或自行编写校准程序
   # 调用 Calibration::calibrate_camera() 方法
   ```
4. 将得到的 fx, fy, cx, cy 值填入 `pipeline.h` 中的 `FOCAL` 常量

### 5.2 瞄具归零校准（5分钟）

**步骤**：
1. 在**恰好15米**处放置靶纸
2. 将枪固定在台钳/支架上，十字准星对准靶心
3. 测量画面中靶心与图像中心的垂直像素偏移
4. 将偏移值填入 `pipeline.h` 中的 `BORE_HEIGHT` 和 `BORE_REF_DIST` 常量

### 5.3 弹道校准（10分钟）

**步骤**：
1. 在10m、20m、30m处各打10发
2. 每种距离测量平均弹着点与瞄准点的垂直偏差(m)
3. 使用 `Calibration::calibrate_ballistics()` 方法计算初速：
   ```cpp
   // 示例调用代码
   autotrigger::BallisticCalibration calib;
   autotrigger::Calibration::calibrate_ballistics(0.32f, 40.0f, calib);
   // calib.muzzle_velocity → 填入命令行 --v0 参数
   ```
4. 启动时使用 `--v0` 参数指定初速：
   ```bash
   autotrigger --v0 70
   ```

## 6. 使用说明

### 6.1 开机流程

```
  ① 插入充电宝 ──▶ 主板启动 (~15秒)
                    │
  ② 等待启动 ──────▶ OLED显示 "AutoTrigger v1.0"
                    │
  ③ 自检通过 ──────▶ 显示十字准星
                    │       █
                    │    ████████
      屏幕显示:     │       █
      十字准星固定   │    ████████
      在屏幕中心     │       █
                    │
  ④ 进入战斗模式 ──▶ 识别人物后显示绿色瞄点
```

### 6.2 屏幕显示解读

```
   ┌────────────────────────┐
   │       ░░░░░░░░░░       │  ← 圆形显示区域
   │    ░░░░░░░░░░░░░░░░    │
   │   ░░░░░░░░░░░░░░░░░░   │
   │  ░░░░░░░░░░░░░░░░░░░░  │
   │ ░░░░░░░░░█░░░░░░░░░░░░ │  ← 十字准星（白色）
   │ ░░░░░░░████████░░░░░░░ │     = 枪口指向
   │ ░░░░░░░░░█░░░░░░░░░░░░ │
   │ ░░░░░░●░░░░░░░░░░░░░░░ │  ← 绿色圆点 = 瞄准点
   │ ░░░░░░░░░░░░░░░░░░░░░░ │     = 补偿后的着弹位置
   │ ░░░░░░░░░░░░░░░░░░░░░░ │
   │  ░░░░░░░░░░░░░░░░░░░░  │
   │   ░░░░░░░░░░░░░░░░░░   │
   │    ░░░░░░░░░░░░░░░░    │
   └────────────────────────┘
```

**操作要领**：
1. 移动枪口，使 **十字准星💢覆盖绿色瞄准点🟢**
2. 当两者重合并持续保持 → 扣住扳机 → **自动激发**
3. 🟢点的位置说明：
   - **在上方** = 目标较远，需要抬高枪口补偿下坠
   - **在左侧** = 目标向右移动，需要提前量
   - **在右侧** = 目标向左移动，需要提前量

### 6.3 战斗中的状态指示

| 屏幕状态 | 含义 | 操作 |
|----------|------|------|
| 🟢 绿点出现 | 已锁定目标 | 移动枪口对准绿点 |
| 🟢 绿点消失 | 目标丢失 | 重新搜索 |
| 🟢 绿点在动 | 目标在移动 | 跟踪绿点方向 |
| 🟢 绿点=准星，3帧 | 🔥自动激发 | 扣住扳机即可 |
| 仅十字准星 | 无目标/系统降级 | 检查传感器 |

## 7. 故障排查

### 7.1 常见问题

| 现象 | 可能原因 | 解决方法 |
|------|----------|----------|
| 开机无显示 | 供电不足/排线松 | 检查USB-C连接,重插排线 |
| OLED花屏 | SPI接线错误 | 对照3.2节接线表重新检查 |
| 不识别目标 | 摄像头镜头脏/选错 | 清洁镜头,确认是62°非鱼眼 |
| 测距不准 | 激光窗口脏/对不准 | 清洁,检查物理安装 |
| 触发信号不输出 | GPIO连接松动 | 用万用表测Pin 11电压 |
| 系统频繁重启 | 电源不稳 | 换充电宝,检查USB线 |

### C. 安全注意事项

- ⚠️ 激光测距仪为 **Class 1** (人眼安全)，**禁止**替换为更高功率激光
- ⚠️ 激发信号为 **3.3V逻辑电平**，勿直接驱动大功率负载
- ⚠️ 室外使用注意**防水**，外壳仅为防溅 (IP44)
- ⚠️ 充电宝请选用**品牌产品**，劣质充电宝可能引发火灾
- ⚠️ **遵守场地规则**，部分真人CS场地可能禁止电子辅助设备
